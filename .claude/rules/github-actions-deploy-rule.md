# GitHub Actions 部署规范规则

## 自动化部署流程

### 工作流配置
- 在.deploy分支上有变化时触发构建
- 使用Hexo生成静态文件
- 将生成的文件部署到master分支
- 确保部署过程中不覆盖重要文件

### Actions 工作流文件
- 位于 `.github/workflows/deploy.yml`
- 包含必要的环境变量设置
- 配置正确的构建依赖项
- 确保安全性（保护敏感信息）

### 部署验证
- 验证构建过程成功完成
- 检查生成的文件完整性
- 确认网站功能正常
- 验证自定义域名配置

### 故障排除
- 记录部署日志以便调试
- 设置适当的错误处理机制
- 确保回滚方案可行
