# 技能文件组织规范规则

## 目录结构
- 每个技能应放在独立的子目录中
- 子目录名称应简洁明了地反映技能功能
- 技能描述文件统一命名为 SKILL.md
- 相关辅助文件可放在同一目录中

## 示例结构
```
.skills/
├── skill-name/
│   ├── SKILL.md          # 技能描述文件
│   ├── script.sh         # (可选)执行脚本
│   └── config.json       # (可选)配置文件
```

## 文件命名
- 主描述文件必须命名为 SKILL.md
- 避免使用特殊字符和空格
- 目录名使用小写字母和连字符分隔
- 保持一致性便于维护

## 内容要求
- SKILL.md 文件应清晰描述技能功能
- 包含使用场景和操作流程
- 列出注意事项和安全措施
- 如适用，提供示例和参数说明
