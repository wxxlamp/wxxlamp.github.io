# 前端开发规范规则

## 主题开发规范

### 代码结构
- 遵循Hexo主题目录结构：layout/, source/, languages/, _config.yml等
- 组件化开发，将功能拆分为独立的partial模板
- 使用语义化的类名和ID命名，采用kebab-case格式

### CSS样式规范
- 使用CSS预处理器（如Stylus/Less），避免纯CSS
- 采用BEM命名方法论或类似模块化命名方式
- 确保样式响应式设计，在不同设备尺寸下正常显示
- 优化CSS性能，减少冗余样式

### JavaScript规范
- 使用现代ES6+语法，但需考虑浏览器兼容性
- 模块化组织代码，避免全局变量污染
- 遵循DOM操作最佳实践，确保性能优化
- 添加适当的错误处理机制

### 性能优化
- 图片懒加载实现
- CSS/JS文件压缩和合并
- 减少HTTP请求数量
- 实现缓存策略

### 测试验证
- 在主流浏览器中测试兼容性
- 验证移动端适配效果
- 确认所有交互功能正常工作
- 检查页面加载速度
