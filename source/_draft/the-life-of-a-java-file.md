1. 对象创建过程：检查(类加载)，分配内存（指针碰撞，空闲列表），对象属性初始化，设置对象头，执行init初始化
 2. DCL是双重检查，常用在单例模式中，设置volatile的原因是synchronized锁住的代码块在new Object的时候会指令重排，具体重排的三步是分配内存空间，实例化对象，把对象引用指向已分配的内存空间。有可能没有实例化对象完成就，另外的线程就指向内存空间
3. 对象的存储布局可包括对象头，示例数据和对其填充
4. 对象头有该对象的hashCode，GC年龄，指向锁记录的指针（轻量级锁），指向重量级锁的指针等
5. 对象定位有两种方式，分别是使用句柄（句柄分别指向对象实例和方法区，实例改变时只会改变句柄的实例指针，更方便）和直接指针（Hotspot使用，访问更快）
6. 如果符合JIT的逃逸分析结果，则对象分配在粘上；如果开启了TLAB，则分配到堆上相应的TLAB上；大对象如数组和长字符串会直接到老年代；其余对象则分配到Eden中
7. 栈中o占8字节（64bit），堆中16字节
